Музыкальная социальная сеть - анализ связей между артистами, жанрами, альбомами и городами

Описание предметной области:
Разработка графовой базы данных для анализа взаимосвязей в музыкальной индустрии. Система отслеживает отношения между музыкантами, их творчеством, географией выступлений и влиянием городов на творческий процесс.

Требования к графовой базе данных:

Таблицы узлов (не менее 4):

Musician (Музыканты)

Genre (Музыкальные жанры)

Album (Музыкальные альбомы)

City (Города)

Таблицы рёбер (не менее 6):

PerformsInGenre (Исполняет жанр)

LivesIn (Проживает в городе)

ReleasedAlbum (Выпустил альбом)

PerformedInCity (Выступал в городе)

CollaboratedWith (Сотрудничал с другим музыкантом)

InspiredByCity (Альбом вдохновлён городом)

Требования к данным:

Каждая таблица узлов содержит не менее 10 записей

Таблицы рёбер содержат реалистичные связи между узлами

Для рёбер PerformedInCity хранить дату выступления

Задачи для решения с помощью SQL-запросов:

5 запросов с использованием MATCH:

Поиск музыкантов по жанру

Определение городов, повлиявших на альбомы определенного жанра

Поиск альбомов музыкантов из конкретного города

Поиск коллабораций музыкантов с выступлениями в одних городах

Анализ жанров альбомов по году выпуска

2 запроса с использованием SHORTEST_PATH:

Поиск кратчайшего пути сотрудничества между музыкантами

Поиск коллабораций в пределах 1-2 шагов

Дополнительные требования:

Визуализация структуры базы данных в diagrams.net

Визуализация графа в Power BI с помощью Force-Directed Graph

Разработка предложения для новой задачи на основе существующей БД

Цель проекта:
Создать систему для анализа музыкальных связей, которая позволяет:

Выявлять скрытые связи между артистами

Анализировать географическое влияние на творчество

Находить потенциальные коллаборации

Исследовать эволюцию музыкальных жанров

Оптимизировать планирование концертных туров
